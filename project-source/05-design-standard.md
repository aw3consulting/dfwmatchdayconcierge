# DFW Matchday Concierge — Design Standard Authority

## Status

This file locks the approved design standard.

It is a governing source-authority file.

## Primary Design Rule

This is not a redesign.

The existing approved DFW Matchday Concierge design standard remains in force.

Implementation work must preserve the approved visual identity, premium private-ride posture, and mobile-first conversion structure.

## Design Objective

The design must make it easy for a guest to quickly understand:

- This is a private SUV ride reservation service.
- The operator uses a 2026 Black Tahoe.
- Availability is limited.
- Pickup + Return is recommended.
- The primary action is Reserve a Ride.
- The service supports airport, stadium, Fan Festival, watch-event, Dallas, Fort Worth, and Arlington ride needs.

## Required Design Posture

The site must feel premium, private, executive, mobile-first, clear, trustworthy, field-ready, simple to use, and booking-focused.

## Approved Visual Direction

Maintain the approved DFW Matchday Concierge direction:

- Navy.
- Gold.
- White.
- Black.
- Premium private Black SUV feel.
- Clean executive layout.
- Strong spacing.
- Large readable CTAs.
- Simple card-based sections.
- High-contrast text.
- Mobile-first readability.

## Prohibited Design Drift

Do not introduce:

- New unrelated visual identity.
- New logo system.
- Generic FIFA-style graphics.
- Official-looking tournament branding.
- Team crest layouts.
- Marketplace-style design.
- Crowded tourism portal design.
- Sponsor-heavy ad design.
- Random color systems.
- Low-contrast text.
- Small unreadable mobile text.
- Overloaded homepage sections.
- Unapproved badge systems.
- Unapproved vehicle art.

## Vehicle Image Rule

Do not use any rejected Tahoe PNG or previously rejected vehicle asset.

If a vehicle image is used, it must be clean, verified, properly cropped, not cutting off content, not interfering with text, not overlapping location names, not low-resolution, not distorted, and approved or safe to use.

If a clean verified vehicle image is not available, omit the vehicle image rather than using a bad asset.

## Protected Marks Rule

Do not use official marks unless legally cleared.

Do not use official FIFA logos, official FIFA World Cup marks, official trophy marks, official team crests, official stadium marks, official venue marks, or lookalike official tournament graphics.

Team names, country names, event names, and venue names may be used in plain text for identification and travel-planning context only.

## Typography Requirements

Mobile readability is mandatory.

Avoid dense paragraphs, cramped cards, long CTA labels that wrap awkwardly, overlapping text, and text over busy images without proper contrast.

## CTA Requirements

Primary CTA:

Reserve a Ride

Primary CTA must be visually dominant.

Secondary CTAs may include View Ride Options, Reserve Pickup + Return, Add Return Trip, Add Airport Return, Add Stadium Ride, Add Fort Worth Ride, Add Dallas Ride, and Request Alternate Driver.

CTA rules:

- Buttons must be large enough for mobile tapping.
- Buttons must not overlap.
- Buttons must not wrap awkwardly.
- Buttons must not be buried under sponsor content.
- Primary CTA must not be visually weaker than secondary actions.

## Page Layout Requirements

Every public page must include a clear headline, short value explanation, primary Reserve a Ride CTA, relevant route/area cards, Pickup + Return prompt where appropriate, limited availability language, and footer disclaimer.

Avoid too many equal CTAs, too many service categories at once, add-ons before route context, sponsor content above booking content, and unverified operational claims.

## Mobile QA Requirements

Every page must pass mobile review for no horizontal scroll, clipped text, cut-off location names, overlapping images, overlapping buttons, hidden form fields, crowded hero section, unreadable sponsor cards, broken CTA, outdated “Book a Ride” CTA, or rejected Tahoe PNG.

Required review widths include iPhone width around 390px, iPhone width around 430px, tablet width around 768px, and desktop width around 1280px.

## Sponsor Design Rules

Sponsor content must not dominate the site.

Sponsor rules:

- No sponsor above the main Reserve a Ride CTA.
- No sponsor popup.
- No sponsor takeover.
- No sponsor should visually compete with the DFW Matchday Concierge brand.
- Sponsor blocks must be labeled.
- Sponsor clicks must not interrupt reservation flow.
- Sponsor design must not imply official event affiliation.

## Form Design Rules

The Reserve a Ride form must be easy to complete on mobile.

Fields must be clearly labeled, required fields clear, conditional fields relevant, error messages understandable, return trip logic visible, limited availability acknowledgment visible, alternate driver permission clear, and submit button prominent.

## Design QA Checklist

Before any public release, confirm approved design standard preserved, no redesign drift, no new unrelated color system, no generic FIFA-style design, no protected marks, no rejected Tahoe PNG, no “Book a Ride” CTA, no cramped mobile sections, no cut-off text, no sponsor above primary CTA, no standalone add-ons without ride context, no unsupported pickup guarantee, and no misleading official affiliation.
