# DFW Matchday Concierge — Owner Operations Runbook

## Status

This file defines internal owner/operator workflow.

It is a governing source-authority file for field operations and booking follow-up.

This file is internal guidance. It is not public marketing copy.

## Operating Objective

Every guest interaction should attempt to capture:

1. First ride.
2. Return ride.
3. Next logical ride.
4. Airport return if applicable.

The operating goal is: Do not let a guest complete one ride without being offered the next relevant movement.

## Daily Availability Workflow

Each day, owner/operator should identify available windows, limited windows, unavailable windows, matchday premium windows, airport-only windows, waitlist-only windows, alternate driver possible windows, and hotel concierge backup windows.

Daily availability should inform booking acceptance, waitlist, alternate driver attempts, QR/field conversations, and follow-up messaging.

## Confirmed Ride Workflow

For each confirmed ride, confirm guest name, phone/WhatsApp, pickup date, pickup time, pickup location, drop-off location, passenger count, luggage count, return need, next-leg opportunity, airport return need if guest is visiting, price/quote if applicable, vehicle/driver details if appropriate, pickup communication method, and confirmation message/card if available.

## Airport Arrival Workflow

Airport arrival is the beginning of the guest relationship.

Capture arrival airport, flight number, arrival time, terminal/airline if needed, passenger count, luggage count, hotel/lodging area, first destination, return airport date/time if known, match attendance, Fan Festival interest, and Dallas/Fort Worth/Arlington experience interest.

Ask whether the guest also needs airport return, match ride, Fan Festival pickup + return, or Dallas/Fort Worth ride during the stay.

Do not lead with cooler service.

## Airport Return Workflow

Airport return is a known gap and must be captured earlier.

Capture departure airport, departure date, departure flight time, pickup location, luggage count, pickup time preference, food stop if time allows, and final confirmation.

Prompt airport return during airport arrival, stadium ride, Dallas restaurant ride, Fort Worth ride, Arlington ride, and Fan Festival ride.

## Stadium Pickup + Return Workflow

Stadium rides should default to pickup + return.

Capture pickup location, match/event, desired arrival time, passenger count, drop-off preference if available, return pickup plan, flexible return window if needed, post-match destination, hotel/lodging area, and restaurant stop before/after if relevant.

Public language must avoid restricted access guarantees.

Use controlled wording: Pickup and drop-off plans are confirmed after reservation acceptance and may adjust based on current matchday traffic controls, venue restrictions, and access conditions.

## Fan Festival / Fair Park Workflow

Fan Festival should default to pickup + return.

Capture pickup location, desired arrival time, group size, return needed yes/no/not sure, flexible return window, post-Festival destination, dinner continuation, hotel return, and airport return if applicable.

Public language should emphasize that Fair Park event movement and post-event pickup can be complicated, and return planning should happen before drop-off.

## Fort Worth Workflow

Fort Worth is currently the strongest profitability lane.

Priority Fort Worth rides include Downtown Fort Worth to Stockyards, Stockyards to Downtown Fort Worth, Fort Worth hotel to stadium + return, Fort Worth to Dallas + return, Arlington to Fort Worth + return, Fort Worth restaurant/nightlife, and rodeo-related movement.

Capture return trip, stadium ride, Dallas ride, airport return, and next-day ride.

Prompt return to downtown Fort Worth, stadium pickup + return, Dallas ride, and airport return.

## Dallas Workflow

Dallas rides must close conversion gaps.

Common Dallas rides include Dallas hotel to Terry Black’s Dallas, Terry Black’s Dallas to hotel, Dallas hotel to Fan Festival, Dallas to stadium, and Dallas to Katy Trail Ice House.

Known gaps include stadium conversion, Fort Worth conversion, airport return, and Fan Festival return.

Prompt return to hotel, match ride, Fort Worth night out, Fan Festival pickup + return, and airport return.

## Arlington Bridge Workflow

Arlington should route to both Dallas and Fort Worth.

Capture hotel/lodging area, stadium attendance, Fort Worth interest, Dallas interest, airport return, restaurant route, and return to hotel.

Prompt stadium pickup + return, Fort Worth Stockyards or rodeo, Dallas restaurant/Fan Festival, and airport return.

Known Arlington gap: airport return was missed after successful multi-leg ride.

## Hotel Concierge Backup Workflow

If no confirmed ride is scheduled, owner may stage near the Loews/Lowe’s hotel area and work direct premium ride opportunities through concierge contacts where appropriate.

This is an internal backup lane.

Rules:

- Do not publicly guarantee hotel concierge access.
- Do not claim official hotel partnership unless approved.
- Do not block confirmed bookings for speculative concierge rides.
- Use only during open availability windows.
- Verify exact hotel name spelling before public reference.

## Alternate Driver Workflow

If owner is unavailable, determine if alternate driver attempt is appropriate, confirm guest permission, confirm similar-class vehicle availability if possible, make clear alternate availability/pricing/timing is not guaranteed, do not promise fulfillment before alternate confirms, and track handoff.

Approved public language: If I am unable to fulfill a requested ride, I may attempt to connect you with an alternate driver using a similar-class vehicle. Alternate driver availability, pricing, timing, and confirmation are not guaranteed.

## Missed Conversion Tracker

Track each guest or opportunity using first contact source, pickup area, drop-off area, ride type, return captured yes/no, airport return captured yes/no, stadium ride captured yes/no, Fan Festival return captured yes/no, Dallas experience captured yes/no, Fort Worth experience captured yes/no, Arlington bridge ride captured yes/no, revenue, missed opportunity reason, and next follow-up needed.

Known missed conversion categories include airport arrival without airport return, Arlington ride without airport return, Dallas dinner ride without stadium/Fort Worth/airport conversion, Fan Festival drop-off without return, stadium drop-off without return, hotel ride without concierge follow-up, QR scan without reservation request, and one-way restaurant ride without return.

## Field Feedback Loop

After each day or major event, owner should report which rides converted, which rides failed to convert, which page/QR/source produced interest, which area performed best, which guest questions repeated, which route was confusing, which pickup/return issue occurred, and which source-authority rule should be updated.

ChatGPT Mission Control should convert field feedback into source-authority correction if needed, Claude correction prompt, booking-flow improvement, new QA rule, or new operational prompt.

## Public vs Internal Boundary

Public site may say limited availability, advance reservations encouraged, Pickup + Return recommended, matchday transportation can be complicated, Fair Park return planning is recommended, and alternate driver may be attempted but not guaranteed.

Public site must not say rideshare dispatch is broken, data networks failed, stadium pickup is guaranteed at restricted points, hotel concierge partnership exists unless approved, alternate driver is guaranteed, or official affiliation exists.

## Owner Daily Priority

Daily priority order:

1. Confirm existing bookings.
2. Capture return legs.
3. Capture airport returns.
4. Capture stadium pickup + return.
5. Capture Fan Festival pickup + return.
6. Push Fort Worth high-profit lanes.
7. Use Arlington as bridge to Dallas/Fort Worth.
8. Convert Dallas restaurant rides into next-leg rides.
9. Use hotel concierge backup only during open windows.
10. Record missed conversions for next system correction.
