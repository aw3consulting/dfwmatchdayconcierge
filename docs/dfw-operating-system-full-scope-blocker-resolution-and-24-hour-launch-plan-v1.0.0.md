## 4. Blocker Resolution Plan

### 4.1 Authority Completeness

Resolution required:

- confirm the required AW3 MKB authority files for this DFW system
- map DFW local files to governing authority stack
- confirm no DFW file outranks latest locked user directive or governing stack
- mark missing authority files as blocking dependencies

Immediate launch posture:

Manual inquiry launch may proceed to approval review only if authority completeness is documented as sufficient for static public launch.

### 4.2 Stale UX Blueprint

Resolution required:

- supersede or replace stale four-lane UX blueprint
- remove retired single-vehicle language
- add seven lanes, dashboards, databases, marketing, SEO, monetization, Oracle OCI, and 24-hour launch requirements

Immediate launch posture:

Old UX blueprint cannot govern production implementation.

### 4.3 Monetization Architecture

Resolution required:

- define monetization by lane
- separate direct service, activation, quote, referral, affiliate, API, and ad revenue
- define ad-free premium zones
- define public fan monetization zones

Immediate launch posture:

Production launch may start with manual quote/activation logic only if public copy does not overstate payment or monetization automation.

### 4.4 Marketing and Acquisition

- coordinator pitch
- VIP pitch
- Media / Corporate pitch
- Fan Hub pitch
- driver recruiting pitch
- concierge recruiting pitch
- first-response scripts
- QR/business-card field-use plan

### 4.5 SEO Minimum

Resolution required:

- production title and meta descriptions
- canonical URLs
- Open Graph tags
- local DFW keyword coverage
- preview indexing decision

Immediate launch posture:

### 4.6 Dashboards

Resolution required:

- driver hub
- concierge hub
- fulfillment queue
- internal inquiry/quote/payment/assignment/closeout board

Immediate launch posture:

### 4.7 Database / Records

Resolution required:

- manual tracker model
- inquiry ID structure
- lane/status fields
- privacy flags
- monetization source

Immediate launch posture:

Private data must not be stored in public GitHub files.

### 4.8 Infrastructure

Resolution required:

- GitHub source-of-truth lane
- GitHub Pages preview lane
- DreamHost static production lane
- Oracle OCI future controlled backend/app/database/dashboard lane

Immediate launch posture:

DreamHost may remain the fastest production path for static launch. Oracle OCI must be tracked for backend and command-center expansion.

### 4.9 Legal / Privacy / Terms

Resolution required:

- independent-service disclaimer

- privacy policy need
- terms need
- payment/refund/cancellation shell
- driver/concierge candidate data boundaries

Immediate launch posture:

Manual call/text/email launch can proceed only with clear public boundaries and no sensitive public-form collection.

### 4.10 Live Route, Mobile, and QR Review

Resolution required:

- review all preview routes
- validate phone/SMS/email links
- validate mobile hierarchy
- validate QR destination path
- confirm no production QR activation before approval
