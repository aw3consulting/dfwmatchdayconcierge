## Mandatory Pre-Code Answer Format

The acting session must return:

1. section name
2. proposed exact copy or copy limit
3. CTA
4. linked destination
5. sponsor handling
6. forbidden content removed
7. pass/fail determination

## Pass Standard

Every section must pass before code begins.

The table passes only if:

1. section count remains limited
2. copy remains within cap
3. concept repetition is controlled
4. homepage routes rather than explains
5. sponsor opportunity is premium and concise
6. details move to linked pages
7. no unsupported claims appear
8. no v1.0.1, v1.0.2, or v1.0.3 failure pattern returns

## Failure Standard

The table fails if any section:

1. duplicates another section
2. contains detailed guide content
3. repeats sponsor or source language unnecessarily
4. creates a directory feel
5. creates a dashboard feel
6. creates a document feel
7. cannot be executed as a premium browser homepage
