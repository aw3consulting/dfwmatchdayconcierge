## Root Cause Determination

The failure occurred because the generation task was interpreted as a visual presentation-board request instead of a controlled file-by-file mockup generation task.

This collapsed a governed deliverable set into a single noncompliant overview artifact.

The failure was not caused by the user instruction. The user instruction required a critical, highly consequential, fully scoped, governed batch. The failure came from execution drift in the assistant workflow.

## Mandatory Pre-Generation Protocol

### Gate 1: File Manifest Gate

The system must produce a file manifest listing every required output file.

For Batch 1, the manifest must list exactly 16 files.

Generation remains blocked until the manifest is explicitly accepted.

### Gate 2: Page Brief Gate

Each file in the manifest must have its own page brief covering page purpose, audience, route, desktop or mobile view, inherited homepage elements, page-specific hero, page-specific planning module, page-specific sponsor inventory, official-source lane, referral or provider handoff lane, premium review boundary, footer and trust structure, and claim controls.

### Gate 3: One-File Generation Rule

Each mockup must be generated as a standalone page view.

One output file equals one mockup.

No prompt may request all pages, the full batch, a board, a grid, a presentation, or a summary view.

### Gate 4: Output Count Gate

After generation, the system must confirm expected file count, actual file count, exact filename list, missing files, extra files, rejected files, and review status.

If the count does not match, the batch fails automatically.

### Gate 5: Readability Gate

Each mockup must be readable at review size.

Any mockup with unreadable microtext, collapsed layout, illegible sponsor inventory, or compressed sections fails automatically.

### Gate 6: Homepage Governance Gate

Each mockup must be checked against the approved homepage system, including dark executive visual system, gold accent hierarchy, header logic, title sponsor inventory area, Advertise With Us CTA, language selector, black and white contrast toggle, page-specific planning module, available sponsor inventory, front-end fulfillment paths, premium reviewed support boundaries, footer distribution, and independent platform disclaimer.

### Gate 7: Ad Inventory Gate

Each page must show page-specific available ad inventory.

Generic sponsor placeholders are insufficient.

Each inventory block must identify placement category, availability status, sponsor CTA, page context, claim-safe status, and visitor-use relevance.

### Gate 8: No Completion Claim Gate

The system may not state or imply that a batch is complete unless every required file exists and is individually listed.
