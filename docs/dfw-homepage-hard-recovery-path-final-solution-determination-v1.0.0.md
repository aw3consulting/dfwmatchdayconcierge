The hard recovery path is the correct final solution before the next homepage prototype attempt.

`The homepage role was not hard-bound before execution.`

The new recovery stack now defines:

1. the v1.0.3 failure
2. the homepage as a minimal premium gateway
3. the exact allowed homepage section structure
4. the boundary between homepage content and linked-page content
5. the rule that detailed content belongs on linked pages
6. the sponsor monetization boundary between homepage, advertise page, sponsors page, and category pages
7. the failure triggers that must block any future homepage prototype that repeats prior drift

## What This Solves

This recovery path solves the central execution problem:

`The homepage can no longer absorb every capability, guide, source, sponsor, provider, or support detail.`

It forces the homepage to serve only six functions:

1. introduce
2. qualify
3. route
4. monetize
5. recognize
6. establish trust
