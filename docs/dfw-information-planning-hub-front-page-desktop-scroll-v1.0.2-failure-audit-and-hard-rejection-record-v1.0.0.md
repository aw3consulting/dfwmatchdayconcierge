## 5. Core Determination

The v1.0.2 homepage desktop scroll mockup is hard rejected.

The file did not correct the v1.0.1 homepage scroll concerns. It moved further away from the required premium visitor-facing website standard.

The failure occurred because the revision treated the previous dead-zone issue as a compression problem instead of a UX hierarchy problem. The result is denser, but less premium, less readable, less emotionally compelling, less browser-realistic, and less aligned with the approved homepage concept.

## 6. Why v1.0.2 Got Worse

### 6.1 Overcompression replaced UX improvement

The prior issue was that the homepage had too many dead zones and felt too document-like.

The correct response was to create better above-the-fold utility, clearer visitor pathways, stronger action hierarchy, and more realistic browser components.

Instead, v1.0.2 compressed large amounts of content into a dense dashboard-style layout.

This created:

1. reduced readability
2. weaker visual hierarchy
3. crowded cards
4. smaller text
5. lower premium feel
6. a layout that feels operational rather than public-facing

### 6.2 Approved homepage concept was weakened

The approved homepage concept had a premium gateway feel built around:

1. strong DFW Matchday brand presence
2. event-energy visual hierarchy
3. clear visitor entry point
4. sponsor visibility
5. guide access
6. reviewed support boundary
7. premium dark executive styling

v1.0.2 reduced the homepage into a compact control panel. It retained some content categories, but it lost the approved homepage's visual confidence and commercial presentation value.

### 6.3 Brand naming drift appeared

The header treatment shifts toward:

`DFW Matchday Information Hub`

The governed public brand direction is:

`DFW Matchday Information + Planning Hub`

This is a brand-identity drift and must not be carried forward.

### 6.4 The page still does not prove clean PDF-to-HTML conversion readiness

A PDF-to-HTML ready mockup must behave like a real browser-rendered page.

The rejected v1.0.2 artifact still shows conversion risk because the visual and parsed order do not confidently reflect a clean semantic browser document. A homepage mockup that cannot preserve proper header, hero, main content, footer, and disclaimer order is not suitable as a governing conversion source.

### 6.5 Sponsor inventory lost premium commercial weight

Sponsor inventory is present, but the commercial surfaces feel like compact dashboard cards instead of premium advertising inventory.

The sponsor system must communicate:

1. value
2. visibility
3. trust
4. availability
5. clear placement logic
6. advertiser confidence

v1.0.2 makes sponsor inventory visible, but not premium enough for the monetization strategy.

### 6.6 Website utility remains under-governed

The page includes fast paths, popular needs, source links, provider handoffs, and reviewed support, but the hierarchy does not guide a visitor quickly enough.

A proper homepage should immediately answer:

1. where do I start?
2. what can I solve?
3. which guide should I open?
4. where are official-source paths?
5. when should I request reviewed support?
6. where can sponsors inquire?

v1.0.2 contains these elements, but the visual hierarchy is too crowded to make them feel intuitive.

## 7. Governance Failure Determination

This failure is not isolated to the generated artifact.

The governing process allowed another homepage mockup revision before the execution method was sufficiently hardened.

The existing standard selected the correct scroll direction, but it did not fully enforce:

1. browser-prototype-first generation
2. no shrink-to-fit PDF scaling
3. native browser viewport capture
4. section-by-section preservation from approved concept
5. above-the-fold utility requirements
6. minimum rendered text size after PDF export
7. semantic read-order validation
8. sponsor inventory value hierarchy
9. UX density balance
10. proof that the mockup can convert to HTML and CSS without redesign

Therefore, governance is part of the failure.
