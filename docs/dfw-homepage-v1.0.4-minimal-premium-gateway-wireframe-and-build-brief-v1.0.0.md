## Build Target

The v1.0.4 homepage must feel like:

`minimal luxury event gateway`

The page must introduce DFW Matchday Concierge, welcome the event audience, provide clear visitor and advertiser paths, recognize approved partners or available partner spaces, and establish trust.

The page must route details to linked pages.

## Approved Component List

Only these homepage components are approved:

1. `SiteHeader`
2. `HeroWelcome`
3. `ServiceSummaryCards`
4. `AudiencePathways`
5. `SponsorPartnerBand`
6. `TrustResponsibility`
7. `SiteFooter`

No additional homepage component may be added unless required for accessibility or validated browser behavior.

## Desktop Wireframe

### Desktop viewport

Use 1440 CSS px width with the governed container system.

### Desktop section order

1. Sticky site header
2. Hero welcome with concise action cluster
3. Service summary card row
4. Audience pathway row
5. Premium sponsor and partner band
6. Trust and responsibility strip
7. Footer and disclaimer

### Desktop layout detail

#### 1. SiteHeader

Layout:

Brand left, concise navigation center, Advertise With Us CTA right.

Allowed navigation:

1. Start Planning
2. Guides
3. Support
4. Sponsors
5. Trust

Header must remain clean and premium.

#### 2. HeroWelcome

Layout:

Two-column premium hero.

Left column:

1. eyebrow
2. main headline
3. supporting headline
4. hero body
5. three CTAs maximum

Right column:

One premium visual/action panel only. The panel may show three compact paths:

1. Visitors
2. Coordinators
3. Advertisers

The right panel must not become a form, dashboard, or guide index.

#### 3. ServiceSummaryCards

Layout:

One compact row or two-row card grid with six cards maximum.

Each card includes:

1. short label
2. one short sentence
3. optional route link

Cards must not contain detailed guide copy.

#### 4. AudiencePathways

Layout:

Five compact audience route cards.

Each card includes:

1. audience label
2. one sentence
3. one CTA

#### 5. SponsorPartnerBand

Layout:

One premium commercial band.

Band includes:

1. Title Sponsor Available
2. Guide and Category Sponsorship Available
3. Partner Recognition Spaces Available
4. Advertise With Us CTA

The band must feel premium, high-value, and commercially credible.

#### 6. TrustResponsibility

Layout:

Four short trust statements in a compact strip.

Allowed statements:

1. Independent Platform
2. Source Distinction
3. Sponsor Disclosure
4. Responsible Planning

#### 7. SiteFooter and DisclaimerBlock

Layout:

Compact footer with concise planning, sponsor, and trust links. Use the locked disclaimer copy.

## Mobile Wireframe

### Mobile viewport

Use 390 CSS px review width and preserve reflow to 320 CSS px.

### Mobile section order

1. Compact header
2. Hero welcome
3. CTA stack
4. service cards
5. audience path cards
6. sponsor and partner band
7. trust statements
8. footer and disclaimer

### Mobile layout detail

1. Header uses brand, compact menu, and accessible advertiser path.
2. Hero stacks copy first, then CTAs.
3. CTA buttons use full-width tap targets.
4. Service cards stack in one column unless two-column cards remain readable.
5. Audience paths stack in one column.
6. Sponsor and partner band remains visually premium and concise.
7. Footer remains compact and does not repeat homepage content.

## Build Acceptance Tests

The v1.0.4 browser prototype passes only if:

1. it feels like a minimal luxury event gateway
2. it uses the locked section order
3. it uses the locked copy model
4. it includes no more than six primary sections before footer
5. it repeats no concept more than twice unless required by navigation, accessibility, footer, or disclaimer
6. it routes detailed information to linked pages
7. it avoids directory, dashboard, document, catalog, and spreadsheet behavior
8. it presents sponsor opportunity as premium and concise
9. it uses no fake partner logos or unapproved sold treatment
10. it preserves independent platform and sponsor disclosure language
11. it works at 1440 CSS px desktop width
12. it works at 390 CSS px mobile width
13. it reflows to 320 CSS px
14. it has no microtext, clipped text, overlap, or horizontal mobile scroll
15. it preserves semantic order from header to disclaimer

## Automatic Failure Triggers

The prototype fails if it includes:

1. planning form as the main homepage structure
2. popular-needs grid that repeats guide categories already shown elsewhere
3. guide gateway that duplicates service summary cards
4. full sponsor inventory detail
5. full official-source section
6. full provider handoff section
7. long trust or legal explanations beyond the locked copy
8. dashboard-style action panel
9. document-like content stacking
10. generic landing page language unrelated to DFW Matchday Concierge
11. any official affiliation implication
12. any live booking, dispatch, payment, AI fulfillment, or general visitor Command Hub claim without approval
