## Core Determination

The approved DFW Matchday Information + Planning Hub homepage mockups must govern every linked `index.html` page before final homepage-to-HTML conversion is authorized.

The linked page mockups are the second largest and most critical visual asset group in the DFW Matchday Concierge launch system. They define the full public experience behind the approved homepage and must be generated as a unified governed visual system, not isolated pages, placeholder screens, generic shells, or loosely related concepts.

## Governing Visual Authority

Every generated page mockup must inherit the approved homepage system:

1. dark executive visual theme
2. gold / champagne accent hierarchy
3. premium command-center structure
4. same header logic
5. same title sponsor inventory model
6. same Advertise With Us business CTA model
7. same language selector placement logic
8. same black / white contrast toggle placement logic
9. same public information-first hierarchy
10. same front-end fulfillment card language
11. same sponsor and featured partner inventory treatment
12. same footer and disclaimer structure
13. same mobile companion logic
14. same rounded card language and spacing system
15. same independent platform claim boundaries

## Non-Drift Rule

Blocked drift includes changing the theme, changing the header, changing sponsor inventory style, replacing public planning with backend dashboards, overpromoting Connect as a visitor CTA, using QR as a main on-site visitor action, showing sold sponsor inventory where inventory is only available, adding unsupported official affiliation language, creating generic directory pages, reducing pages into shells, omitting page-specific ad inventory, omitting relevant context, omitting mobile structure, omitting claim language, or introducing unrelated logo treatments.

## Output Scope

This package governs 16 linked `index.html` pages across the full linked page system.

Each linked page requires:

1. one desktop mockup
2. one mobile mockup
3. page-specific sponsor inventory

## Universal Page Composition Standard

Every linked-page mockup must include header, page hero, context module, front-end fulfillment cards, ad inventory and sponsor system, and footer.

Header must include approved logo placeholder position, `DFW Matchday`, title sponsor inventory area, Advertise With Us CTA, language selector where appropriate, and black / white contrast toggle.

Each page hero must include page-specific title, visitor-use promise, information-first positioning, primary route-specific CTA, secondary guide/planning CTA, and sponsor-safe inventory area where appropriate.

Each page must explain who it is for, what decision it helps make, what information is provided, what external paths may be shown, and when premium review is appropriate.

Every page must include status-safe cards such as Information Only, Official Source, Sponsored Placement, Featured Partner, Affiliate / Referral, Third-Party Provider, Operator Review Required, Quote Required, Subject to Confirmation, and Restricted.

Unsold inventory must appear as available, not sold.

Accepted language includes Available Sponsor Placement, Category Available, Featured Partner Slot Available, Reserve This Category, Sponsor This Guide, Title Sponsor Available, Featured Placement Available, and Local Partner Slot Available.

## Pre-Generation Quality Gates

Before any mockup generation begins, the system must confirm page purpose, audience, ad inventory, required context, CTA hierarchy, claim controls, official-source lane where relevant, sponsor/partner/referral path where relevant, premium review boundary, approved homepage visual governance, mobile variant requirement, and page membership in the full information and planning hub system.

## Approval Process

Each generated page mockup must receive one of the following statuses: Approved, Approved with minor corrections, Conditional pass, Revision required, Rejected, or Superseded.

Only approved or explicitly accepted conditional mockups may move forward into HTML conversion alignment.

## HTML Conversion Boundary

Final homepage HTML conversion remains blocked until the linked-page mockup set is generated and approved.

Allowed before approval: component planning, route mapping, shared layout rules, design token extraction, content architecture, data model refinement, and sponsor inventory mapping.

Blocked before approval: final homepage HTML conversion, final route-link deployment, final public page completion claim, DreamHost production deployment, QR activation, and treating linked routes as finished pages.
