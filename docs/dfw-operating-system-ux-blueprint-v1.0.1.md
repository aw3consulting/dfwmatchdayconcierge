# DFW Operating System UX Blueprint v1.0.1

## 1. Purpose

This file is the active UX blueprint for DFW Matchday Concierge.

It replaces `docs/dfw-operating-system-ux-blueprint-v1.0.0.md`.

## 2. Active UX Standard

DFW Matchday Concierge must present as a DFW World Cup Matchday Operating System, not a transportation-only website.

The UX must support public inquiry, manual operator review, command hub preview, activated planning, quote/scope review, payment/deposit status, confirmation, assignment, fulfillment, and closeout.

## 3. Active Lanes

The UX must include seven lanes:

1. VIP Guest Concierge
2. Team Family Support
3. Media / Corporate
4. Team Fan Hubs
5. Driver Onboarding
6. Concierge Onboarding
7. Fulfillment Layer

## 4. Active Operating Chain

Required UX chain:

1. Public Site
2. Inquiry
3. Verification
4. AI-Assisted Intake Preview
5. Operator Review
6. Command Hub Preview
7. Activation
8. Itinerary
9. Quote / Scope
10. Payment / Deposit Status
11. Confirmation
12. Assignment
13. Fulfillment
14. Completion / Closeout

No page may imply a later state before the required prior state is complete.

## 5. Active Vehicle Standard

The active vehicle language is:

`Black Executive Vehicle Class`

No public UX may present a guaranteed specific vehicle make or model before confirmation.

## 6. Homepage UX

Homepage must include:

- approved logo
- premium navigation
- DFW Matchday Operating System positioning
- Start Inquiry CTA
- seven-lane selector
- AI-assisted intake preview panel
- map/location shell
- command hub preview CTA
- driver onboarding CTA
- concierge onboarding CTA
- fulfillment CTA
- claim/status badge system
- independence disclaimer

## 7. Connect / QR UX

Connect page must include:

- live call path
- live SMS path
- live email path
- lane selector
- what-to-send guidance
- operator review boundary
- no payment, dispatch, routing, assignment, or fulfillment guarantee

## 8. Command Hub UX

Command hubs must be dashboard-like and state-aware.

Required command hub concepts:

- VIP Guest Command Hub
- Team Family Support Command Hub
- Media / Corporate Command Hub
- Team Fan Hub
- Personal Driver Command Hub
- Concierge Operations Command Hub
- Fulfillment Routing Hub
- Operator Fulfillment Queue

## 9. Intake and Profile UX

Required intake/profile concepts:

- AI Concierge Discovery + Intake
- lane-specific intake pages
- driver onboarding
- concierge onboarding
- fulfillment routing intake
- VIP Guest Profile
- Team Family Support Profile
- Media / Corporate Profile
- Team Fan Hub Profile
- Driver Profile
- Concierge Profile
- Fulfillment Request Profile

## 10. SEO UX Minimum

Production candidate pages must include:

- title tag
- meta description
- canonical URL
- Open Graph title
- Open Graph description
- Open Graph type
- local DFW keyword coverage where appropriate

## 11. Monetization UX

The UX must distinguish:

- free public guidance
- activated planning
- quote-based support
- referral / third-party paths
- affiliate or sponsored paths
- direct service path
- information-only path
- restricted path

Ads may appear only in approved public zones. Premium private hubs remain ad-free unless explicitly approved.

## 12. Dashboard and Data UX

The UX must support dashboards and manual recordkeeping even before backend implementation.

Minimum launch dashboard can be a private manual tracker outside the public repo.

No private client, driver, concierge, payment, or sensitive data may be stored in public GitHub files.

## 13. Infrastructure UX

GitHub Pages remains preview only.

DreamHost is the immediate static production target after approval.

Oracle OCI is the future or parallel controlled path for backend, database, dashboard, command-center app, API, AI automation, and secure operator tooling.

## 14. Claim Boundaries

The UX must not claim:

- official tournament, team, venue, federation, sponsor, or credential affiliation
- guaranteed fulfillment
- guaranteed specific vehicle assignment
- live AI automation
- live payment collection
- live dispatch
- live routing automation
- automatic driver assignment
- automatic concierge assignment

## 15. Final Determination

This file is the active UX blueprint for implementation and launch-readiness review.

It governs future frontend, mockup, SEO, dashboard, data, and production-candidate work until superseded.
